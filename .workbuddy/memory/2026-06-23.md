## 媒体上传问题修复（核心进展）

### 问题1: alt_text NOT NULL 约束违反
- **现象**: 上传图片显示"数据库记录保存失败: null value in column 'alt_text'"
- **原因**: API 路由中 `alt_text: altText || null`，用户不填替代文本时传 null
- **修复**: 改为 `alt_text: altText || file.name`，用文件名作为兜底
- **影响文件**: `src/app/api/media/upload/route.ts`, `src/app/admin/(protected)/media/actions.ts`

### 问题2: PDF/视频上传"网络错误"
- **原因**: Vercel 平台请求体限制 4.5MB（免费版和 Pro 版），PDF/视频超限被拒绝
- **方案**: 客户端直传 Supabase Storage
  - 新增 `src/lib/supabase/browser.ts` — 浏览器端 Supabase Client
  - 重写 `UploadForm.tsx` — 文件直接从前端传到 Storage，绕过 Vercel
  - 新增 `src/app/api/media/register/route.ts` — 轻量元数据注册接口（仅 JSON，无文件）
  - 旧路由 `/api/media/upload` 保留但不再使用
