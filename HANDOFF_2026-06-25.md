## 八、关键代码说明

### 1. 后台布局 (`AdminShell.tsx`)
- **当前版本**: flex 布局，导航栏固定左侧 (w-60)
- **内容区**: 默认 `max-w-3xl` 约束
- **作品编辑页**: 使用 `calc(-50vw + 50%)` 突破约束实现全宽

### 2. 作品编辑页 (`works/[id]/page.tsx`)
- **布局**: 三列 grid (`1fr | 720px | 1fr`)
  - 第1列: 空白 (实现居中)
  - 第2列 (红框): 标题 + 文案 + VisualBlockEditor
  - 第3列 (蓝框): 辅助面板 (预览/媒体/分类/设置/版本历史)
- **注意**: 顶部遮挡问题需用 `paddingTop: 7rem` 补偿

### 3. 内容块系统 (`WorkContentBlocks.tsx`)
支持 6 种块类型：
- `text` - 文本块 (支持 left/center/right 对齐)
- `media` - 媒体块
- `gallery` - 图库块 (支持 1/2/3/4 列)
- `video` - 视频块
- `before_after` - Before/After 对比块
- `pdf` - PDF 块

### 4. Server Actions (`actions.ts`)
所有后台操作使用 Server Actions：
- 无需客户端状态管理
- Zod schema 验证输入
- `requireAdmin()` 鉴权
- `revalidatePath` 刷新页面

## 九、常见问题与解决方案

### 1. 构建失败
**原因**: TypeScript 类型错误  
**解决**:
```bash
npm run build 2>&1 | head -50  # 查看详细错误
```
常见问题：
- 未使用的导入
- 类型不匹配
- 缺少 null 检查

### 2. Supabase 连接失败
**原因**: 环境变量未设置或错误  
**解决**:
- 检查 `.env.local` 是否存在
- 确认 `NEXT_PUBLIC_SUPABASE_URL` 和 `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` 正确
- 运行 `npm run cms:check` 检查连接

### 3. 图片不显示
**原因**: Supabase Storage 权限或 URL 错误  
**解决**:
- 检查 Storage bucket `portfolio-media` 是否存在
- 确认 RLS 策略允许公开访问
- 使用 `src/lib/cms/media-url.ts` 构建 URL

### 4. 部署后功能异常
**原因**: 数据库迁移未执行  
**解决**:
- 检查 `supabase/migrations/` 中哪些文件未执行
- 在 Supabase Dashboard 手动执行迁移
