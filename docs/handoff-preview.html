2.3 作品上传流程（参考站酷）

  • 三段式向导：媒体上传 → 元数据 → 内容块编辑
  • 拖拽上传、多选文件
  • 支持 JPG/PNG/GIF/MP4/PDF 等格式
  • 单文件最高 10GB
  • 上传后自动生成内容块

2.4 状态与版本控制

  • 作品状态：draft / published / private
  • 私密分享：生成 preview token，仅链接可见
  • 自动版本归档：每次修改作品后自动保存快照
  • 定时发布：scheduled_publish_at + 手动/自动发布机制

九、常见问题与注意事项

  1. Supabase 项目 ID 是 hnujowombcgfxledpnxe，所有 SQL 必须在该项目上执行。
  2. 管理员角色：Auth 用户 app_metadata 中必须包含 "role": "admin"（直引号）。
  3. 软删除：所有 delete 操作都使用 deleted_at 字段，不物理删除。
  4. 静态回退：当 Supabase 不可用时，前台自动回退到静态数据。
  5. 编码问题：如果中文显示为问号，检查 SQL 编码和 hasEncodingCorruption 检测逻辑。
