## 四、版本控制系统

### 架构

```
用户操作（保存/编辑/删除块）
    ↓
autoArchiveAfterChange()  ← 静默自动归档当前状态
    ↓
archiveWorkVersion()      ← 生成版本快照（JSONB）
    ↓
work_versions 表          ← 存储 version_number + snapshot + label
```

### 核心文件

| 文件 | 用途 |
|------|------|
| `src/lib/cms/versions.ts` | 版本控制核心逻辑（归档/列表/回滚/快照） |
| `src/components/admin/VersionHistoryPanel.tsx` | 版本历史 UI 组件 |
| `src/app/admin/(protected)/works/actions.ts` | 版本控制 Server Actions（第 1433-1646 行） |

### 版本控制 API

```typescript
// 归档当前作品状态为新版本
archiveWorkVersion(client, workId, adminUserId?, label?, source?: "auto"|"manual"): Promise<number | null>

// 列出所有历史版本（标记当前版本）
listWorkVersions(client, workId, limit?): Promise<WorkVersionListItem[]>

// 获取特定版本快照
getWorkVersionSnapshot(client, workId, versionNumber): Promise<WorkVersionSnapshot | null>

// 回滚/前进到指定版本（先自动备份当前状态）
rollbackWorkVersion(client, workId, targetVersionNumber, adminUserId?): Promise<{ savedVersion, restoredVersion }>
```

### Server Actions

| Action | 功能 |
|--------|------|
| `archiveWorkVersionAction` | 手动保存版本（带 label） |
| `rollbackWorkVersionAction` | 回滚到指定版本 |
| `restoreForwardWorkVersionAction` | 前进到更高版本 |
| `listWorkVersionsAction` | 列出版本（直接调用，非 FormData） |

### 自动归档触发点

所有破坏性操作均会触发 `autoArchiveAfterChange`：
- `updateWork`（更新作品元数据）
- `updateWorkMedia`（更新作品媒体）
- `updateWorkTaxonomy`（更新分类标签）
- `createTextBlock` / `updateTextBlock`
- `createMediaBlock` / `updateMediaBlock`
- `createGalleryBlock` / `updateGalleryBlock`
- `createVideoBlock` / `updateVideoBlock`
- `createPdfBlock` / `updatePdfBlock`
- `createBeforeAfterBlock` / `updateBeforeAfterBlock`
- `deleteWorkBlock`
- `createBlockDirect` / `updateBlockDirect` / `deleteBlockDirect`

### 2. work_versions 表缺少 label 列

版本自动归档会静默失败（`archiveWorkVersion` 插入 `label` 列但表无此列）。

**解决**：执行 `20260623120000_add_label_to_work_versions.sql`。

### 3. Auth 角色元数据编码问题

Supabase auth 表的 `raw_app_meta_data` 中 `"role": "admin"` 如果使用了弯引号，`private.is_admin()` 会匹配失败。

**排查**：
```sql
SELECT id, email, raw_app_meta_data FROM auth.users WHERE email = '3624457672@qq.com';
```

**修复**：
```sql
UPDATE auth.users SET raw_app_meta_data = jsonb_set(raw_app_meta_data, '{role}', '"admin"') WHERE email = '3624457672@qq.com';
```

### 4. 邮箱不一致

项目中有两个邮箱：
- `3624457672@qq.com` — 管理员登录
- `hello@sscyl.top` — 公开联系邮箱

设计上无冲突，但如果需要统一请排查 `src/data/portfolio.ts`、`src/components/home/CapabilityBands.tsx`、`src/app/api/contact/route.ts`。

### 5. 联系邮件未配置

`RESEND_API_KEY` 和 `RESEND_FROM_EMAIL` 未配置，联系表单提交后不会发送邮件通知（仅存入数据库）。

## 十二、架构约定

1. **Server Actions + FormData 模式**：所有后台操作走 Server Actions + FormData，无客户端状态管理
2. **软删除策略**：所有表 delete 操作设置 `deleted_at`，不物理删除
3. **静态回退**：Supabase 不可用时自动回退到 `src/data/portfolio.ts` 的静态数据
4. **RLS（Row Level Security）**：所有表启用 RLS，通过 `private.is_admin()` 函数控制写权限
5. **版本快照**：JSONB 存储作品+内容块完整状态，支持回滚和前进
